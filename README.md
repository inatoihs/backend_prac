## 概要

このプロジェクトは、Go 言語で作成された Web アプリケーションを Docker コンテナで実行し、MySQL データベースと連携させるものです。

Goアプリケーションは HTTP エンドポイントを提供し、MySQL データベースから現在の時間を取得して表示します。パフォーマンスチューニングのために意図的に効率の悪い実装が含まれています。

## 特徴

- Docker を使用して複数のサービス（Go アプリケーション、MySQL、Nginx）をコンテナ化
- MySQL データベースに接続し、クエリ結果を表示
- パフォーマンスチューニングを学ぶためのサンプルデータと効率の悪いクエリ

## セットアップと実行

### 前提条件

- Docker と Docker Compose がインストールされている必要があります。

### ステップ 1: プロジェクトのクローン

```bash
git clone https://github.com/your-username/your-repository.git
cd your-repository
```

### ステップ 2: 必要なデータベースの初期化

プロジェクトディレクトリ内の `mysql/init.sql` に初期化スクリプトが含まれています。Docker Compose を使用して MySQL コンテナを起動すると、このスクリプトが自動的に実行されます。

### ステップ 3: Docker コンテナのビルドと起動

```bash
docker-compose up --build
```

これで、`nginx` コンテナがポート 80 でリクエストを受け付け、`go` コンテナのアプリケーションにプロキシします。`go` コンテナはポート 8080 で待機し、MySQL から現在の時間を取得して表示します。

### ステップ 4: アプリケーションへのアクセス

ブラウザまたは curl を使用して以下の URL にアクセスします:

```bash
curl http://localhost
```

ブラウザでアクセスすると、MySQL データベースから取得した現在の時間が表示されます。

## パフォーマンスチューニング

このプロジェクトでは、以下をはじめとするパフォーマンス問題を意図的に含めています：

- N+1 クエリ問題
- 大量データの読み込み
- インデックスの欠如
